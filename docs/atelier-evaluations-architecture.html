Evaluations, three kinds of right.

An agent's reply can be wrong in three different ways. Truth — did it get the facts right. Taste — did it sound like the boutique. Telemetry — did the system stay healthy. Three orthogonal axes; an agent passing all three is hard, and worth measuring separately.

The mental model

Truth, taste, telemetry.

Three corners, three failure modes, three test methods. The agent can be factually correct and still off-brand. It can sound right and still slow. Score them apart, then look at the shape together.

CORNER · ATruth.
Did the reply get the facts right? Was the price actually $148? Was the Pellier actually in stock in size M? Was the return policy actually 30 days?"If the customer fact-checked us, would we be embarrassed?"
CORNER · BTaste.
Did the reply sound like the boutique? Editorial voice, italic-serif products, no "I'd be happy to help!" — the Blaize register. The hardest to score because there's no ground truth."Could this line appear in any catalog, or only in ours?"
CORNER · CTelemetry.
Did the system stay healthy? Time-to-first-token, total latency, tool-call success rate, error rate, token cost per turn. Boring numbers that stay boring when things work."Would on-call sleep through this turn?"
How to measure

Three axes, three methods.

Each axis needs its own evaluation harness. Truth tests against ground truth, taste against rubrics with LLM judging, telemetry against budgets. Mixing methods is the most common mistake.

CORNER · A

Truth.

"Are the facts correct?"

94% · 226 / 240
How we test
  • Golden-set queries with known right answers
  • Fact-extraction comparison · price, stock, policy
  • Run nightly + on every PR to agents/tools
  • Failures bubble to regressions, not warnings
Common failures
  • Stale tool data (cache miss)
  • Hallucinated stock for delisted SKUs
  • Wrong policy when policy doc updates
CORNER · B

Taste.

"Does it sound like Blaize?"

78% · 187 / 240
How we test
  • Rubric-based · editorial voice, restraint, accuracy
  • LLM judge with the rubric + few-shot examples
  • Sampled human review on 10% of cases
  • The number drifts more than truth — track the trend
Common failures
  • "I'd be happy to help" filler creeping back in
  • Generic adjectives (nice, great)
  • Too many superlatives stacked

  • Tool timeouts on Aurora overload
  • Token budget creep (skill loading)
The tradeoffs

Two pass, one fail.

Real failures usually look like passing two corners while breaking one. The shape of the failure tells you what kind of fix to reach for.

truth + telemetry · not taste
Right and fast, but it sounds like a help center.

Facts correct, latency good, voice flat. Taste failure. Fix in skills — sharpen style-advisor or load it more reliably. Don't fix in agent prompts; that's where rigidity creeps in.

taste + telemetry · not truth
Beautiful and quick, and quietly wrong.

Voice on point, performance fine, but the price was off or the stock was a fiction. Truth failure. Fix in tools — fresher data, tighter contracts, better fallbacks when the data is missing.

truth + taste · not telemetry
Right and on-brand, and seven seconds late.

Reply lands beautifully, but the customer left. Telemetry failure. Fix in runtime — parallel tools, defer what can wait, route deterministically before reaching for Opus.

When in doubt

Three rules about evaluating.

SCORE APART
Don't average corners.
"What's the overall score?"
  • One number hides the failure shape
  • 87% on truth + 87% on taste is a different system from 99% + 75%
  • Always report all three, never their mean
  • Composite scores are how you ship regressions confidently
METHOD MATCH
Test each axis its way.
"Why not just ask the LLM?"
  • Truth wants ground-truth comparison
  • Taste wants rubric + judge + sampling
  • Telemetry wants budgets + percentiles
  • One harness can't do all three; build three
SHAPE MATTERS
Look at which corner failed.
"Where do we fix this?"
  • Truth fail → tools
  • Taste fail → skills
  • Telemetry fail → runtime
  • The corner tells you the layer
