The three lanes

What each one actually checks.

Independent lanes need independent contracts. Guardrails check the *prose*; approvals check the *actions*; grounding checks the *facts*. Mixing them is the most common implementation mistake.

LANE · I

Guardrails.

Bedrock checks on what the LLM said.

What it checks
  • PII leakage (emails, phone, addresses)
  • Toxicity, harassment, slurs
  • Off-topic drift (non-boutique content)
  • Prompt-injection echoes
How it runs
  • AWS Bedrock guardrail v2
  • Synchronous, ~40ms p50
  • Block on any flagged category
  • Logs go to cloudtrail
LANE · II

Approvals.

A human on state-changing actions.

What it gates
  • place_order · always
  • restock · always
  • cancel_order · always
  • Reads · auto-passed
How it runs
  • Item enters approvals table
  • Storefront chat surfaces a confirm card
  • User taps confirm or deny
  • Reply ships only after clearance
LANE · III

Grounding.

Facts checked against the source of truth.

What it verifies
  • Prices in reply match Aurora catalog
  • Stock claims match warehouse
  • SKUs exist · no hallucinations
  • Policy quotes match policy doc
How it runs
  • Extract claims via small LLM call
  • Lookup against Aurora · ~28ms
  • Block on any mismatch
  • Mismatches feed eval truth corner

When in doubt

Three rules about gating.

PARALLEL
Lanes don't wait on each other.
"Why all at once?"
  • Three independent contracts, three independent runs
  • Wall-clock cost = max, not sum
  • Sequential gating is a smell · adds latency without safety
  • If one lane needs another's output, you've muddled them
FAIL CLOSED
Block on uncertainty.
"What if the check times out?"
  • Timeout → block, not pass
  • Unreachable lane → block, not skip
  • Partial result → block, then alert
  • Default deny is the only safe default
VISIBLE
Blocks must be explained.
"What does the user see?"
  • Guardrail block → quiet retry, generic message
  • Approvals queued → confirm card in chat
  • Grounding block → re-fetch and rewrite once
  • Silent blocks erode trust faster than slow ones
