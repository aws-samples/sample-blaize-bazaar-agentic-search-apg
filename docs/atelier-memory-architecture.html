The two tiers

The flow

One turn, step by step.

A request arrives. The agent doesn't ask both stores for everything. It walks down the cheapest readable surface first and pays for depth only when the turn earns it.

i.

Open the session.

The turn arrives with a session_id already established. AgentCore opens the STM buffer for this session — a thin RPC that returns instantly if the session is warm.

STM
# request enters with session
session = agentcore.open(
  session_id="sess_4f",
  customer_id="cust_a3",
)
ii.

Read short-term context.

Pull the last twelve turns from STM. Always. This is the conversation thread — it's cheap and gives the agent the immediate context it needs to interpret the new message.

STM
stm = session.get(
  limit=12,
)
# → list[Message]
iii.

Decide if long-term is needed.

If the conversation references something outside the recent buffer — "like the linen shirt I bought last fall", "my usual size" — the agent reaches into LTM. Otherwise, skip. The check is a small classifier call, ~120ms.

STM
if needs_ltm(
  message=msg,
  stm_context=stm,
):
  recall() # step iv
iv.

Recall from long-term.

Embed the user's message, query Aurora pgvector for the top-k most similar facts scoped to this customer. Returns past purchases, preferences, prior conversations — whatever the embedding decided was relevant.

LTM
emb = embed(msg)
ltm_facts = aurora.find_similar(
  emb=emb,
  customer_id=session.cust,
  k=5,
)
v.

Compose, then write back.

The agent reasons with both contexts in scope, composes a reply, and appends the new turn back to STM. The new exchange is now part of the next turn's short-term context — and, after a debounced background job, makes its way into LTM if it produced a durable fact.

Both
reply = agent.compose(
  stm, ltm_facts, msg
)
session.append(msg, reply)
# LTM write happens async
When in doubt

Two tiers, three questions.

SHAPE
What is it?
"How is the data structured?"
  • STM · ordered list of turns
  • LTM · vector index of facts
  • STM holds prose, LTM holds extracted nouns
  • STM is the conversation; LTM is what the conversation produced
SCOPE
Whose memory?
"Who can read this?"
  • STM · scoped to one session
  • LTM · scoped to one customer across sessions
  • STM dies when the session ends
  • LTM persists across visits
COST
When to pay?
"Is this worth a read?"
  • STM · always read, ~47ms
  • LTM · read conditionally, ~312ms
  • The classifier deciding "needs LTM?" is itself ~120ms
  • Don't reach for LTM by default — earn it
