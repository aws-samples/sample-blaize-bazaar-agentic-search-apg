Three rules

How the agent talks to the schema.

READS
Through tools, never direct.

The agent never writes raw SQL. Every read goes through a typed tool — search_products, get_order_history. The tool owns the query; the agent owns the question.

# tool layer holds the SQL
def search_products(q: str):
  return db.fetch(
    "SELECT … FROM products …"
  )
WRITES
Gated, and confirmed.

Domain writes (orders, returns, restocks) require explicit user confirmation in the chat. The agent proposes, the customer confirms, the tool commits. Session writes are ambient and idempotent.

if user_confirmed:
  db.execute(
    "INSERT INTO orders …"
  )
SESSION
The bridge, kept thin.

The session table is what makes the agent's runtime durable. Cart contents, turn count, last-active stamp. Written end-of-turn, read at boot — small, hot, indexed.

session.upsert(
  cart=cart,
  turn_count=n,
)
