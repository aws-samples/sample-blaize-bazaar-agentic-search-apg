When in doubt

Three rules about granting.

DEFAULT DENY
Bind less, not more.
"Should this agent really need it?"
  • New tool starts with zero grants
  • Each grant is added explicitly, with a reason
  • Access creep happens silently — review periodically
  • Capability sprawl is a risk surface, not a feature
ROLE FIT
Tool to job, not to convenience.
"Is this central to the agent's role?"
  • Recommendation owns get_recommendations
  • Support reads it; doesn't own it
  • Multiple owners is a smell — pick one
  • If two agents fight over a tool, you have a missing agent
GATE SIDE EFFECTS
Writes need confirmation.
"Does this change something durable?"
  • Reads are free; writes are gated
  • restock requires explicit user "yes"
  • The gate is at the tool, not the agent
  • Don't trust an agent to ask the question on your behalf
